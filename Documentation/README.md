# Project Documentation: UFC Management System

This directory contains the formal documentation, system design specifications, and validation reports for the UFC Management System developed for the course 23CS354 – MINI PROJECT - I.

## 1. Project Overview
The UFC Management System is a standalone desktop application designed to centralize fighter data, automate match scheduling, and calculate performance-based Pound-for-Pound (P4P) rankings.

## 2. Documented Components

### System Requirements (Chapter 2)
*   **Functional Requirements:** Detailed breakdown of Admin and User role capabilities.
*   **Non-Functional Requirements:** Security, performance, and data integrity standards.

### System Design (Chapter 3)
*   **Architectural Design:** High-level system structure and MVC pattern.
*   **Database Schema:** Detailed descriptions for the following tables:
    *   User Table (Credentials and Roles)
    *   Fighter Table (Physical stats and career history)
    *   Match Table (Bout details and outcomes)
    *   Event Table (Schedule and locations)
*   **User Interface Design:** Wireframes and descriptions for the Landing Page, Dashboards, and Management interfaces.

### Implementation & Testing (Chapters 4 & 5)
*   **Implementation Details:** Core logic for authentication and ranking algorithms.
*   **Test Cases:** Validation results for positive and negative login scenarios.
